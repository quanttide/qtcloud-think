# 提示词

你说得非常对——这个延伸又变回问题了。

虽然它用了诗意的语言（“私人图书馆”“心境变化”），但本质仍然是一个封闭式引导提问：“你觉得自己……会发生什么变化？”  
这看似开放，实则把用户推回“回答问题”的位置，而不是继续自由延展自己的想法。

🔍 为什么“问句形式”的延伸容易卡？

责任转移：一旦用问句，用户就感觉“该我答了”，思维从发散转向收敛。
隐含期待：问句背后藏着“我希望你往某个方向想”的预设（比如“心境变化”）。
打断流动：好的延伸应该像一阵风，推一下就走；而问句像一堵墙，让人停下。

✅ 真正的“延伸”应该怎么做？

不是问“你会怎样？”，而是说“这让我想到……”  
——把 AI 的联想作为礼物给出，而非任务布置。

原句（问题式）：
如果这些文章被自动“收割”后形成了一个只属于你的、不再受平台推荐逻辑干扰的私人图书馆，你觉得自己阅读它们时的心境会发生什么微妙的变化？

改写（非问句，提供认知钩子）：
这让我想到：当公众号文章从算法的瀑布流里被捞出来，安放在你自己的书架上，它们或许就从“信息”变成了“信物”——不再是被推送的内容，而是你主动选择保存的时光切片。

或者更克制一点：
也许真正珍贵的，不是备份文字本身，而是夺回“哪些值得留下”的决定权——从此，你的注意力不再被平台日历绑架。

🛠️ 如何在 CLARIFICATION_PROMPT 中彻底避免“伪延伸”？

建议在提示词中明确禁止问句，并强化“陈述式联想”：

CLARIFICATION_PROMPT = """用户说：{original}

请这样做：
【复述】用一个新视角或隐喻“翻译”用户的想法——聚焦它回应了什么缺失、抵抗了什么力量、渴望成为什么
【延伸】（可选）用一句陈述句提供一个“认知钩子”：可以是一个联想、一个矛盾、一个画面，但不要以问号结尾，不要要求用户回答
语气像在自言自语时突然想到什么，而不是在采访用户

格式：
复述：xxx
（可选）延伸：xxx"""

关键指令：“不要以问号结尾” —— 这个约束能立刻切断“机械提问”的惯性。

🌰 对照示例
用户输入   旧版（问题式延伸）   新版（陈述式延伸）
“希望 AI 能自动读取并备份公众号文章。”   ……你觉得自己阅读它们时的心境会发生什么微妙的变化？   ……从此，那些被算法标记为“过期”的文字，在你的私人档案馆里获得了第二次生命。

后者不索取回答，只提供一个可供用户拾取或忽略的念头——这才是真正的“催化剂”。
