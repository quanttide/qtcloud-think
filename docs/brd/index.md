# 量潮思考云 - 业务需求文档

## 场景一：AI辅助对话与信息筛选

### 工作角色

- 用户：与AI对话，筛选有价值的信息
- AI：生成回复，学习用户偏好

### 问题

触发：用户与AI进行多轮对话，希望保留有价值的标注。

现状：用户说一句话，AI回复。用户需要从AI输出中选择值得保留的标注，不相关的内容通常直接丢弃。

困难：每次都要手动筛选，工作繁琐。AI不了解用户的筛选标准，下次还会输出类似的不相关内容。

后果：用户需要反复筛选同样的问题，效率低。

### 假设

如果存在这样一个平台，能够记录用户的筛选行为，AI从中学习用户的偏好，调整输出策略，用户就能减少筛选工作。

## 场景二：反馈闭环

### 工作角色

- 用户：提供反馈
- AI：从反馈中学习

### 问题

触发：用户丢弃了AI的某些输出。

现状：用户选择保留的标注通常是有价值的，不保留的是不相关的。AI无法感知这些反馈。

困难：用户没有明确告诉AI为什么丢弃，AI无法从中学到规律。

后果：AI持续输出用户不需要的内容，用户体验下降。

### 假设

如果存在这样一个平台，能够让AI从用户的筛选行为中推断偏好，形成反馈闭环，AI就能越来越懂用户。
